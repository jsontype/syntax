[Docker]

1. 도커란?

1. 도커 실행과 종료
  실행 : docker-compose up -d
  종료 : docker-compose down

1. Dockerfile 작성
  프로젝트 루트 디렉토리에 Dockerfile을 생성한다.
베이스 이미지, 작업 디렉토리, 의존성 설치, 파일 복사, 실행 명령 등을 정의합니다.
도커 이미지 빌드
터미널에서 docker build -t [이미지이름]:[태그] . 명령어로 이미지를 빌드합니다.
도커 컨테이너 실행
docker run -p [호스트포트]:[컨테이너포트] [이미지이름]:[태그] 명령어로 컨테이너를 실행합니다.
컨테이너 확인
docker ps 명령어로 실행 중인 컨테이너를 확인합니다.
이미지 배포 (선택사항)
Docker Hub에 계정을 만들고 로그인합니다.
docker push [사용자이름]/[이미지이름]:[태그] 명령어로 이미지를 배포합니다.
이 순서를 따라가면 기본적인 도커 사용법을 실습해볼 수 있습니다. 실제 개발 환경에서는 더 복잡한 설정이나 최적화가 필요할 수 있지만, 이 과정으로 도커의 기본 개념과 워크플로우를 이해할 수 있습니다.
