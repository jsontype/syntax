[Alias]

1. Alias
  Alias란 
  터미널에서 alias는 자주 사용하는 긴 명령어나 일련의 명령어를 짧은 단어나 문구로 대체할 수 있도록 해주는 단축 커멘드이다.
  이는 특히 복잡하거나 입력하기 번거로운 명령어를 간단하게 실행할 수 있게 만들어준다.

1. Alias 사용시의 주의사항
  Alias를 추가할 때는 반드시 기존에 사용되고 있는 Alias와 겹치지 않는지 한번 입력해서 확인해보고 추가해야 한다. 아래는 그 예시이다.
    $ add 터미널에 입력
    "zsh: command not found: add"이 뜨는지 확인 후 add alias를 추가한다.
  Alias는 잘못 삭제하면 쉘 자체에 좋지 않은 영향을 줄 수도 있으므로 모르는 alias는 함부로 삭제하지 않도록 주의한다.



[자주 쓰는 Alias 목록]

1. Alias 추가 : 터미널에서 사용시 현 세션에서만 적용되며, 영구적 추가는 .zshrc를 수정해야 한다.
  $ alias chrome='open -na "Google Chrome" --args --user-data-dir=/tmp/dev_chrome_profile --disable-web-security'
  $ alias fet='git fetch origin'
  $ alias pul='git pull origin'

1. Alias 삭제 : 현 세션에서만 적용되며, 영구적 삭제는 .zshrc를 수정해야 한다.
  $ unalias pul

1. Alias 리스트 보기
  $ alias



[Zsh에서 영구적인 Alias 설정]

1. ~/.zshrc 수정
  $ code ~/.zshrc : zsh 설정 파일을 열어준다.
  파일의 맨 아래에 아래와 같이 원하는 alias 정의를 추가한다. (주의 : Private은 개인용으로, 현장에서는 쓰지 않는다.)
  ```
    # Public Alias
    alias chrome='open -na "Google Chrome" --args --user-data-dir=/tmp/dev_chrome_profile --disable-web-security'
    alias fet='git fetch origin'
    alias pul='git pull origin'
    alias add='git add .'
    
    # Private Alias
    alias commit='git commit -m "study"'
    alias push='git push origin master'
  ```
  $ source ~/.zshrc : 변경 사항을 적용한다.
