    추가적으로, 나중에 SSR을 구현하게 될 시 서버에서 들고 있는 데이터를 쉽게 보존하여 브라우저에게 전달 하게 되는 과정이 훨씬 쉬워지기도 한다.

    사실 API 연동할 때, 실무에서는 주로 Redux, Redux 미들웨어를 함께 사용하여 구현한다. 이유는 다음과 같다.
    - Redux를 사용할 경우, 더 체계적으로 비동기 관련 액션들을 관리할 수 있다.
    - Redux-saga, Redux-observable 같은 미들웨어 라이브러리들을 사용할 경우, 복잡한 비동기 작업을 조금 더 편하게 처리할 수 있다.
    따라서, Context-api은 Redux 없이도 API 연동을 구현할 수 있다는 정도로 기억해두자.

1. API 구현을 하기 위해 알아둬야할 것
    먼저, 비동기 처리에 대한 기본적인 개념(Promise 와 async/await의 기본적인 개념)을 알아야한다.
    모른다면 JS에서 Promise 관련 부분을 학습하고 오자.
