[개발환경 (Local, Dev, Stage, Production)]

1. Local Environment (Local env) : 로컬 환경
    먼저 개발을 하려면, 각자 개발자 PC에 개발 및 테스트 환경이 셋업 되어 있어야 한다.
    각 개발자의 PC마다 설치된 "서버 환경"을 local 환경이라고 한다. (예. 각 PC에 개발툴과, 컴파일러 등이 설치되어 있는 환경)
    Local 환경을 구축할시에 가장 주의해야 할점은 모든 개발자가 같은 개발 환경을 사용해야 한다는 것이다. (Node 버전을 nvm을 사용해서 통일, Tomcat등의 자바 서버, MySQL등 DB 버전등을 통일)
    같은 환경이 아니라면 정작 코드를 합칠때, local에서 잘 작동했던 코드가 작동하지 않는 경우가 많다. 
    개발 환경을 표준화 하는 방법은 여러 방법이 있다. (Maven으로 JDK와 라이브러리 버전 지정, Docker를 사용 등)

1. Dev Environment (DEV env) : 개발서버 환경
    DEV 환경은 개발자들이 각각 만든 코드들을 합쳐서, 서버 환경에서 테스트해볼 수 있는 환경이다.
    소스코드를 형상관리 시스템에 commit하면, 코드는 이 DEV 환경에 자동으로 배포되고, 이 환경에서 테스트가 된다. 
    기능 개발을 위주로 하기 때문에, 서버의 환경은 PRO 환경보다 훨씬 작다. 
    예를 들어 PRO 환경이 클러스터링 환경으로 수개의 서버로 구성된다면, DEV 환경은 한 두 개의 서버로 기능 구현이 가능한 정도로 구축한다.
    기능의 버그는 대부분 이 선에서 수정한다.

1. Staging Environment (STA env) : 스테이징 환경
    운영 환경과 거의 동일한 환경을 만들어 놓고, 운영환경으로 이전하기 전에, 여러 가지 비기능적인 부분(Security, 성능, Stress 테스트, Batch 테스트, 장애등)을 검증하는 환경이다.

1. Production Environment (PRO env) : 운영 환경
    실제 서비스를 위한 운영 환경이다.
