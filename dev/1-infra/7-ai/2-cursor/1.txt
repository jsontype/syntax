[Cursor AI 활용 방법]

1. Tab 자동완성 (가장 자주 씀)
  코드 입력 중 Tab 으로 AI 제안 수락
  한 줄·한 블록씩 빠르게 작성할 때 사용
  Settings → Cursor Tab 에서 on/off 가능

1. Inline Edit (선택 영역 수정)
  수정할 코드 블록 드래그 → Ctrl+K (macOS: Cmd+K)
  "이 함수 에러 처리 추가해줘" 처럼 짧은 지시
  해당 범위만 바로 수정

1. Chat (질문·설명·단일 파일)
  Ctrl+L (macOS: Cmd+L) 로 사이드 Chat 열기
  코드 이해, 버그 원인 파악, 리뷰 요청에 적합
  사용 예시:
    - 이 컴포넌트 props 흐름 설명해줘
    - 이 에러 메시지 원인이 뭐야?
    - @src/api/auth.ts 이 파일 리뷰해줘

1. Agent (여러 파일 수정·자동 실행)
  Ctrl+I (macOS: Cmd+I) 또는 Chat에서 Agent 모드
  파일 생성·수정, 터미널 명령, 테스트까지 연속 작업
  사용 예시:
    - 로그인 API 연동하고 폼 컴포넌트까지 만들어줘
    - 이 버그 고치고 관련 테스트도 돌려줘
  변경 전 diff 확인 후 Accept / Reject



[컨텍스트 지정 (@ 멘션)]

1. Chat / Agent 입력창에서 @ 로 범위 지정
  @파일명 : 특정 파일만 참조 (예: @src/app/page.tsx)
  @폴더명 : 폴더 전체 참조 (예: @src/components)
  @Codebase : 프로젝트 전체 검색 후 관련 코드 참조
  @Docs : 공식 문서 참조 (Next.js, React 등)
  @Web : 웹 검색 (최신 API·에러 해결)
  @Git : 최근 변경 diff 참조
  범위를 좁힐수록 답변·수정이 정확해짐



[프로젝트 설정]

1. Rules 작성 (.cursor/rules 또는 .cursorrules)
  프로젝트 루트에 규칙 파일을 두면 매 요청마다 컨텍스트로 적용
  # Project Context
  - Tech stack: Next.js 15, TypeScript, Prisma, PostgreSQL, Tailwind
  - 주요 폴더: app/, components/, lib/, api/
  - 컨벤션: 함수형 컴포넌트, named export, 2 spaces
  - 금지: console.log 남기기, any 남발
  - 명령어: pnpm dev, pnpm build, pnpm test

1. .cursorignore 작성 (선택)
  node_modules, .env, 빌드 산출물 등 AI가 읽지 않을 경로 지정
  불필요한 컨텍스트·민감 정보 노출 방지



[Agent Git 커밋 설정]

1. Co-authored-by 제거 (Commit Attribution OFF)
  Agent가 커밋할 때 Co-authored-by: Cursor trailer가 붙는 것을 끄려면:
  Cursor Settings (VS Code 설정 아님) → Agents → Attribution → Commit Attribution: OFF
  Ctrl+Shift+P → Cursor Settings 로 열기
  PR에도 안 붙이려면 PR Attribution: OFF
  CLI Agent: %USERPROFILE%\.cursor\cli-config.json 에 commitAttribution: false

[Cursor 기능 개요]

1. Cursor에서 할 수 있는 것
  Tab 자동완성으로 코드 한 줄·한 블록 빠르게 작성
  Inline Edit (Ctrl+K)로 선택 영역만 수정
  Chat (Ctrl+L)으로 질문·설명·리뷰
  Agent (Ctrl+I)로 여러 파일 수정·터미널 실행·테스트까지 연속 작업
  @ 멘션으로 파일·폴더·Codebase·Docs·Web 컨텍스트 지정
  Plan / Ask / Debug 모드로 작업 방식 전환
  .cursor/rules, Skills, MCP 로 프로젝트·팀 규칙·외부 도구 연동
  Agents Window (Cmd+Shift+P → Agents Window)에서 병렬 에이전트 관리
  변경 diff 확인 후 Accept / Reject

1. Agent 모드별 역할
  Agent : 파일 수정·터미널·테스트까지 자율 실행 (기본)
  Ask : 읽기 전용 탐색·질문 (/ask)
  Plan : 코딩 전 설계·접근 방식 논의 (/plan)
  Debug : 버그 원인 추적·가설 검증



[Cursor 슬래시 명령어]
참조: https://cursor.com/docs/cli/reference/slash-commands
  Agent Chat 입력창에서 / 입력 → 명령 목록 팝업
  /summarize 등 일부 명령은 목록에 안 보일 수 있음 → /summarize 처럼 직접 입력

1. 모드 전환
  /plan : Plan 모드 (구현 전 설계)
  /ask : Ask 모드 (읽기 전용 탐색)

1. 세션·대화 관리
  /new-chat : 새 채팅 시작
  /summarize : 대화 요약·컨텍스트 압축 (메시지 1개 이상 있을 때)
  /compress : /summarize 와 동일 (CLI)
  /resume [folder] : 이전 채팅 이어하기

1. 모델·실행 설정
  /model [name] : 모델 설정·목록
  /max-mode [on|off] : Max mode 토글 (지원 모델)
  /auto-run [on|off|status] : Agent 자동 실행 토글
  /sandbox : Sandbox·네트워크 접근 설정

1. Git·병렬 작업 (Agents Window / 최신 버전)
  /worktree : 별도 git worktree 에서 격리 작업
  /best-of-n : 여러 모델에 같은 작업 병렬 실행 후 결과 비교

1. Rules·Commands·Skills
  /rules : Rules 생성·편집 (.cursor/rules, User Rules)
  /commands : 커스텀 슬래시 명령 생성·편집
  커스텀 명령 저장 위치:
    프로젝트: .cursor/commands/[command].md
    전역: ~/.cursor/commands/[command].md
  Skills: .cursor/skills/ 또는 ~/.cursor/skills/ → / 로 Skills 호출

1. MCP
  /mcp list : MCP 서버 목록·설정
  /mcp enable [name] : MCP 서버 활성화
  /mcp disable [name] : MCP 서버 비활성화

1. 상태·설정·기타
  /usage : 사용량·streak 확인
  /help [command] : 명령 도움말
  /about : 환경·CLI 설정 정보
  /vim : Vim 키 토글
  /setup-terminal : 터미널 키바인딩 자동 설정
  /copy-request-id : 마지막 request ID 복사
  /copy-conversation-id : conversation ID 복사
  /feedback : 피드백 전송
  /logout : Cursor 로그아웃
  /quit : 종료
