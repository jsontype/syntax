[Git 실습순서]

1. 프로젝트 시작
    github 가입
    slack 설치
    git 설치
    git config --list user.name과 user.email 설정
    git 프로젝트에 username으로 초대받고 참가하기
    slack 프로젝트 채널에 초대받고 참가하기
    git clone 따기

1. 이슈 대응 해보기 (리더1 : jsontype, 개발자1 : azumaapp, 개발자2 : 학생)
    리더1이 초기 폴더구성 : 
        리더1이 initial 커밋
        (api 취득 상태, 폴더구성 완료 상태에서 master에 푸시)
    리더1이 개발자1에게 이슈 주고 대응하는 것 시연 : 
        리더1이 개발자1에게 App Title의 컴포넌트화를 이슈로 날림
        개발자1이 대응하고 PR날리고 슬랙 메시지까지 시연
    개발자2가 이슈 대응 : 
        이슈 1개 대응해보기
        리더1이 이슈 배정하고 슬랙날리고, 개발자2가 PR날리고 슬랙 메시지까지 보내보기
    개발자2가 이슈 2개 동시 대응 :
        이슈 2개 동시 대응해보기
        (add, commit은 이슈별로 따로따로 할 것)
    숙제 : 개발자2가 작업중 마스터와 컨플릭트 발생했을 시의 대응 : 
        개발자2가 이슈 2개중 1개 하는 동안, 리더1이 두번째 이슈를 동시에 작업하기
        리더1이 개발자2가 보고있는 소스코드를 건든 상태로 먼저 푸시하기
        개발자2가 PR 날릴 때 컨플릭트 해결해보기
    숙제 : 개발자2가 실수로 잘못된 브랜치에 커밋한 것을 취소하고 브랜치 변경해서 다시 작업 대응 :
        이슈 2개중 1개는 정상 작업, 2번째는 실수로 1번째 작업시의 브랜치에서 커밋하기
        git reset HEAD^ 후 push -f 해보기
