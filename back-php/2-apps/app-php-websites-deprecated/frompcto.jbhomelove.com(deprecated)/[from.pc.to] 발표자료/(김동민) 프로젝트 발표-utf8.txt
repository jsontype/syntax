페이지별 발표 원고

#1
안녕하세요. 저희는 양동화 팀의 발표자 김동민이라고 합니다.
저희 팀은 PL에 양동화, 백엔드 김동민, 프론트엔드 조승우 씨가 맡고 있습니다.
그럼 지금부터 프로젝트 발표를 시작하겠습니다.

#2 
먼저, 프로젝트의 개요를 소개하겠습니다.
(PPT 내용읽기)

#3
분담 역할은, 먼저
양동화 리더는 기본코딩부터 시작해서, 
전체적인 기본설계, 상세설계, 팀원간 역할 조율,
로직, 매소드 부분인 Model, Controller, Dispatcher 부분을 담당했습니다.
김동민 본인은 백엔드 개발자로서, 
MySQL 데이터베이스의 SQL문을 이용해 Mapper 부분을 처리했습니다.
또한 프로젝트의 발표를 맡았습니다.
조승우 프로젝트원은 발표 PPT 제작하셨고,
개발과정에서는 HTML, CSS, Javascript를 사용해 전체적인 화면을 조성했습니다.

각자의 기호에 맞게 역할 분담이 잘 이루어졌고,
결합과정에서도 큰 문제 없이 잘 작동하는 것을 볼 수 있었습니다.
디버깅은 수강신청부분에서만 2회 정도 발생해서 매우 원활하게 진행되어
1월말~2월초에 벌써 프로젝트가 끝나버릴 정도로 빠르게 진행되었습니다.

#4 
다음으로 IDE입니다. 저희는 서브라임텍스트와 ATOM 두가지를 사용했는데,
가장 주력으로 사용한 것은 ATOM 1.43.0입니다. 
개발 언어는 PHP 7.2, 데이터베이스는 MySQL 5.7을 사용했습니다.

#5
데이터의 공유 툴은 SVN과 Git 중에서 Git 을 채택하였습니다.
Git의 장점 중 하나인 Branch를 4회 사용함으로써 버전간의 이질감 없이 진행하였습니다.

위 화면은 테스트 Git과 Branch를 사용한 증거이며, 각각의 Git 코드들을 보면
해당 버전에서의 작업상황을 자세히 볼 수 있게 깔끔하게 정리되어 있습니다.
아래 그래프를 보시면 세 팀원간의 Git Push비율이 고르게 분포되어 있음을 확인할 수 있습니다.
팀원간 역할 분담의 원활도를 느낄 수 있습니다.


#6
팀원간의 소통은 Slack으로 하였습니다.
이곳에서 Git이나, 개발에 참고할 URL, 그리고
요건정의서, 기본설계서, 상세설계서 등의 문서를 주고받았습니다.

#7 
다음은 프로젝트 메인화면입니다.
Modal 기능을 사용해 각각의 "자세히 알아보기" 버튼을 클릭시 팝업이 뜨게끔 하는 기능을 첨가해서 화면의 이동없이 해당 페이지 내에서 관련 소개 내용을 볼 수 있게 하였습니다.
이 Modal 기능은 자바스크립트 프레임워크인 JQuery을 이용하였습니다.


#8
최신 강의 목록입니다.
리더의 베이스 코딩과 메인 설계서를 토대로 조승우 팀원이 화면을 제작해주었습니다.
서로간의 확실한 의사소통과 명료한 설계로 정확히 원하는 화면을 만들어주셨습니다.


#9
로그인 설계서와 실제 구현된 화면입니다.
여기서 로그인 멤버 부분의 SQL은 본인이 코딩하였고,
리더가 코딩해주신 로그인, 회원가입, 수정, 탈퇴 등의 멤버 로직과 연결하여
세션을 구현하였습니다.
뿐만 아니라, MD5 암호화를 구현하여
보이지 않는 부분의 "보안" 문제까지 잡아주었습니다.
이 상태에서 화면만 조금 더 업그레이드하게 된다면,
이는 거의 실무 레벨급의 코딩이라고 생각합니다.


#10 
회원가입화면과 해당 화면 설계서 입니다.
이 부분 역시 리더의 설계서를 토대로 제가 데이터베이스 SQL안의 회원가입 컬럼을 설정하고 도입하였습니다.
왼쪽을 보시면,
다양한 컬럼이 있는데,
ID, 이름, 닉네임, 이메일주소, 주소는 변동텍스트 데이터타입인 varchar를 사용하였고,
화면상에서는 전부 인풋 타입 텍스트로 구현했고, 주소만 텍스트에어리어를 사용했습니다.
또한 모든 항목에 자바스크립트를 이용해 태그 금지 코딩을 씌워서, 
XSS, 즉 크로스사이트스크립팅 방식의 해킹을 방지하였습니다.
생년월일, 자택전화번호, 휴대전화번호는 int를 사용해, tag나 string 입력을 방지하였습니다.
성별은 화면상에서는 라디오 인풋을 도입, DB상으로는 고정텍스트 데이터타입인 char를 사용했습니다.


#11
강사소개 부분은 정적 웹페이지로 조승우 팀원이 만들어주었습니다.

강사 소개 밑에는 해당 강사가 참여한 강의가 출력되도록 하였습니다.

추후 유료 강의를 제작할 경우 유료 수강신청을 할 수 있게
코딩을 해놓고, 결제 모듈을 주석화하였습니다.

#16
저희 사이트를 자평하도록 하겠습니다.
첫째, 실제 PG들에게 도움을 줍니다.
저희에게 가장 필요한 것은 "코딩 실력"입니다.
코딩의 실력을 가늠할 때
코딩의 실행은 50%만 이해해도 가능하지만
그 개념을 가르치는 것은 90% 이해해야지만 가능하다고 합니다.
저희는 실력 향상을 할 수 있는 강의 사이트를 제작하면서, 
부족한 사람들은 강의를 들으면서 공부를 할 수 있고,
저희는 강의를 제작하면서 90%의 이해를 하려고 노력했습니다.
그래서 서로 Win-win 할 수 있는 어플리케이션을 만드려고 시도했다는 점에서
저희 스스로 만족스러운 결과가 나왔다고 생각하고 있습니다.
